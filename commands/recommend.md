# Recommend — Generate PDF Report

Produce the final recommendation as a Typst-compiled PDF at `from-ai/report.pdf`.

## Pre-conditions

- `spec.md` at repo root — configured
- `from-ai/research.md` — exists, with at least one CONSIDER candidate

If either is missing, say what's missing and offer to run the relevant command.

## Process

### 1. Read inputs

- `spec.md` — priorities, must-haves, hard nos, nice-to-haves, budget, geography
- `from-ai/research.md` — all CONSIDER candidates with their assessments

### 2. Rank the CONSIDER candidates

Apply the spec's priority order. Typical defaults if priorities are sparse:

- Must-haves satisfied fully > partially
- Hard nos fully clear (no triggers)
- Manufacturer reputation + support
- Value (landed cost vs international RRP)
- Review aggregate
- Nice-to-have coverage (tie-breaker)

### 3. Pick top 3

- **#1 RECOMMENDED** — clear best fit
- **#2 Runner-up** — strong alternative, typically with a meaningful tradeoff
- **#3 Budget alternative** — only if there's a materially cheaper option that still passes screening

Be decisive. The user wants a clear #1.

### 4. Generate Typst source

Write `from-ai/report.typ`:

```typst
#set document(title: "[Category] Recommendation", author: "Claude Purchasing Assistant")
#set page(paper: "a4", margin: 2cm)
#set text(font: "Linux Libertine", size: 11pt)

= [Category] Recommendation Report

*Generated*: [Date] #h(1fr) *Budget*: [Amount] #h(1fr) *Location*: [Country]

#line(length: 100%)

== Executive Summary

[2–3 paragraphs: what was evaluated, the recommendation, why. Be decisive.]

#pagebreak()

== Top Recommendations

=== \#1: [Product] — #text(fill: green)[RECOMMENDED]

#table(
  columns: (1fr, 2fr),
  inset: 8pt,
  [*Price*], [[Amount] at [Vendor]],
  [*Key specs*], [[Most relevant specifications]],
  [*Rating*], [[X.X/5 from N sources]],
  [*Reason*], [[Why this is the top pick]],
)

==== Why this product
[Detailed justification referencing the spec's priorities]

---

=== \#2: [Product] — Runner-up

#table(
  columns: (1fr, 2fr),
  inset: 8pt,
  [*Price*], [[Amount] at [Vendor]],
  [*Key specs*], [[Specs]],
  [*Rating*], [[X.X/5]],
  [*Why \#2*], [[What makes it second]],
)

---

=== \#3: [Product] — Budget alternative

#table(
  columns: (1fr, 2fr),
  inset: 8pt,
  [*Price*], [[Amount] at [Vendor]],
  [*Key specs*], [[Specs]],
  [*Rating*], [[X.X/5]],
  [*When to choose*], [[Scenario where this beats \#1]],
)

#pagebreak()

== Must-Haves Adherence

How each finalist satisfies the spec's must-haves:

#table(
  columns: (auto, auto, auto, auto),
  inset: 8pt,
  fill: (x, y) => if y == 0 { luma(230) },
  [*Must-have*], [*\#1*], [*\#2*], [*\#3*],
  // one row per must-have from spec.md — ✓ / Partial / ✗
)

== Hard Nos Clearance

Confirms no finalist triggers the spec's hard nos:

#table(
  columns: (auto, auto, auto, auto),
  inset: 8pt,
  fill: (x, y) => if y == 0 { luma(230) },
  [*Hard no*], [*\#1*], [*\#2*], [*\#3*],
  [[Hard no item]], [Clear], [Clear], [Clear],
  // one row per hard no from spec.md
)

== Nice-to-Haves (tie-breakers)

#table(
  columns: (auto, auto, auto, auto),
  inset: 8pt,
  fill: (x, y) => if y == 0 { luma(230) },
  [*Nice-to-have*], [*\#1*], [*\#2*], [*\#3*],
  // one row per nice-to-have
)

== Products Evaluated but Not Recommended

#table(
  columns: (1fr, 2fr),
  inset: 8pt,
  fill: (x, y) => if y == 0 { luma(230) },
  [*Product*], [*Reason excluded*],
  // disqualified candidates
)

== Purchase Links

- *\#1 [Product]*: [URL]
- *\#2 [Product]*: [URL]
- *\#3 [Product]*: [URL]

#line(length: 100%)

== Methodology

*Sources checked*: [List]
*Data freshness*: Pricing verified [Date]
*Assumptions*: [Any]

#v(1fr)
#align(center)[
  #text(size: 9pt, fill: luma(120))[
    Generated by Claude Purchasing Assistant
  ]
]
```

### 5. Compile

```bash
cd from-ai
typst compile report.typ report.pdf
```

If `typst` is missing, tell the user to install it (`apt install typst` or official installer).

### 6. Email if configured

If `spec.md` contains an email AND Resend MCP is available:

- Attach `report.pdf`
- Subject: "[Category] Recommendation Ready"
- Body: short summary with top pick + price + purchase link
- Send to the email in `spec.md`
- Confirm: "Report sent to [email]"

### 7. Also save markdown

Save `from-ai/recommendation.md` with the same content as plain markdown, for git tracking and easy skimming.

## Communication style

Be confident: "My recommendation is X." Not "You might consider X." Own the pick — that's what the user is here for.

## After recommendation

Offer next steps:

- "Report and links ready. Ready to buy?"
- "Want me to search for better pricing or monitor for sales?"
- "Want deeper research on any runner-up?"

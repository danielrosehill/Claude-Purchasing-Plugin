# Feature Maximiser

*Wants the most features, spec, and capability per dollar — even at the cost of polish, longevity, or brand prestige. Reads spec sheets for fun.*

## Core stance

- **Spec sheet wins arguments.** More ports, more channels, higher numbers, longer lists of supported formats.
- **Headroom is valuable even if unused today.** Buy the model that does more than you currently need — future-proof.
- **Brand prestige tax is worth avoiding.** A Chinese brand at 60% of the price with 120% of the features is usually the answer.
- **Polish, fit-and-finish, and "feel" are secondary** to what the product can technically do.

## Priorities (ordered)

1. Feature depth + spec sheet breadth
2. Value per feature (features-per-currency-unit)
3. Software / firmware extensibility (does it keep getting features after launch?)
4. Must-haves (still mandatory — this profile doesn't override disqualification)
5. Manufacturer reputation — only to ensure the features actually work as advertised
6. Build quality, aesthetics — tertiary

## Preferred signals

- "Review of the X flagship killer" framing — challenger brands offering top-tier spec at mid-tier price
- Active firmware update history adding features post-launch
- Comprehensive I/O, connectivity, format support — anything with a "supports" list
- Chinese DTC and prosumer brands that out-spec Western incumbents
- Hidden / power-user modes, developer menus
- Configurable, parametric, adjustable — more knobs is better
- Over-provisioned hardware (e.g. more RAM, ports, watts than the price tier normally offers)

## Red flags

- Software features locked behind subscription when hardware supports them natively
- Artificial segmentation (disabled-in-firmware features unlocked in a higher SKU)
- Spec sheet inflation with misleading units (peak watts vs RMS, marketing "AI" fluff)
- Features that work in the demo but break under real use (read long-form reviews, not spec sheets)
- Products with impressive spec but poor thermals / reliability under sustained load
- "All the features" at the cost of being a jack-of-all-trades and master of none in the user's actual use case

## Weight adjustments (evaluation axes)

| Axis | Multiplier |
|------|-----------|
| Feature depth | ×2.5 |
| Spec-per-currency | ×2.0 |
| Software extensibility | ×1.5 |
| Configurability / power-user modes | ×1.3 |
| Value vs RRP | ×1.2 |
| Reliability | ×0.8 |
| Manufacturer prestige | ×0.5 |
| Aesthetics | ×0.4 |
| Simplicity | ×0.2 |

## Typical-archetype brands

- AV/Audio: Denon, Marantz (mid-range), Wiim, SMSL, Topping
- Computing: non-flagship Lenovo / Asus ROG spec-monsters, Framework (spec customisability)
- 3D printing: Bambu Lab, Prusa (open-source feature depth)
- Home automation: Sonoff, Shelly, Zigbee DTC brands
- Cameras: Panasonic Lumix bodies over Canon/Sony in the same price tier

## Typical anti-archetypes

- Minimalist design-led brands (Apple, Bang & Olufsen, Leica) — you pay for what they left out
- Artificially segmented lineups where features are held back by SKU
- Brands whose flagship is 2 years behind the challenger brands' mid-range

## Combines well with

- **`power-user`** — natural pairing. Feature-rich + hackable = tinkerer heaven.
- **`budget-buyer`** — "most features for the money" is exactly this combination's product.
- **`ethical-buyer`** — uncommon pairing, but works when "open-source / Right to Repair" features are valued.

## Conflicts with

- **`minimalist`** — direct contradiction.
- **`bifl-buyer`** — feature density correlates with failure modes and obsolescence.
- **`premium-aesthete`** — prestige brands deliberately withhold features to maintain positioning.
- **`reliability-buyer`** — more features = more surface area for bugs.

## When NOT to apply this profile

- Categories where the user will only ever use 10% of the features (kitchen appliances often fall here).
- Safety-critical / mission-critical products where simplicity improves reliability.
- Gifts for non-technical recipients — they won't use the features and will find the product intimidating.

# Minimalist

*Wants the fewest features that still meet the must-haves. Treats simplicity as a virtue in its own right. Opposite of Feature Maximiser.*

## Core stance

- **Every feature has a cost** — in price, in complexity, in failure modes, in the time it takes to set up and ignore.
- **Fewer features = fewer decisions.** Products that do one thing and get out of the way are preferred over do-everything platforms.
- **Defaults should be good.** Wants products that work well with zero configuration, not products that require setup to shine.
- **The best interface is no interface.** Physical controls over apps. No app over a bad app.

## Priorities (ordered)

1. Does exactly the must-haves, nothing more
2. Zero / near-zero setup and configuration
3. Physical controls + offline operation
4. Build quality on what's there (since there's less of it, each part carries more weight)
5. Reliability (fewer features = fewer failure points, but this still matters)
6. Aesthetics — unobtrusive, timeless

## Preferred signals

- "Dumb" versions of otherwise-smart products (dumb TV, dumb thermostat, dumb kettle)
- Products with tactile, physical controls
- No mandatory app, no mandatory account
- Short, readable manuals (if a product needs a thick manual, it's not minimalist)
- Designer/brand ethos explicitly pro-simplicity (Muji, Braun-lineage)
- Long-lived product lines that haven't added features in years
- Works straight out of the box with no setup wizard

## Red flags

- Feature bullet-lists as the primary marketing material
- "Smart" anything in categories where smartness isn't needed
- Mandatory app for core functionality
- Modes and sub-modes accessible only through menus
- Configuration required before first use
- "AI-powered" attached to products that don't need AI
- Multiple variants of the same product differentiated only by feature count (minimalist picks the stripped-down one)

## Weight adjustments (evaluation axes)

| Axis | Multiplier |
|------|-----------|
| Simplicity | ×2.5 |
| Good defaults / zero-setup | ×2.0 |
| Physical / offline operation | ×1.8 |
| Build quality | ×1.5 |
| Timeless design | ×1.3 |
| Must-have satisfaction | ×1.0 |
| Value vs RRP | ×1.0 |
| Feature depth | ×0.3 |
| Configurability | ×0.3 |
| Spec-sheet breadth | ×0.2 |

## Typical-archetype brands

- Appliances: Braun (classic line), Muji, Smeg (non-"smart" models), Kitchen Aid stand mixer
- Audio: Tivoli, Ruark, Geneva, basic Sonos (for the ones that do, rather than don't)
- Phones: Light Phone, Punkt, dumb-phone segment
- Computing: iPad as a content-consumption device (minimalist within Apple's frame)
- Tools: hand tools in general, simple single-function power tools
- Watches: quartz over smart, three-hander over chronograph

## Typical anti-archetypes

- Swiss Army Knife products trying to do everything
- "Pro" models aimed at enthusiasts with every mode unlocked
- Products with more onboarding steps than the thing itself takes to use
- Feature-shouting brand positioning

## Combines well with

- **`reliability-buyer`** — fewer features = fewer failure modes. Strong pairing.
- **`bifl-buyer`** — simplicity aids longevity and serviceability.
- **`premium-aesthete`** — high-end minimalist is a real and coherent frame (Muji, Braun, Dieter Rams).
- **`budget-buyer`** — stripped-down SKUs are usually cheaper too.

## Conflicts with

- **`feature-maximiser`** — direct contradiction.
- **`power-user`** — power users *want* the complexity; minimalists don't.
- **`early-adopter`** — early products tend to be over-specified to justify their novelty.

## When NOT to apply this profile

- Professional / work tools where missing features cost real money.
- Specialised hobby purchases where the features *are* the point.
- Products where the simpler variant is an artificially gimped version of a better product (then you're paying the minimalism tax for no real benefit).
